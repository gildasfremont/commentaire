# Instructions pour Claude Code

## Comment travailler sur ce repo

1. Lis tes issues dans Linear (projet "Commentaire MVP", team "Projets avec Claude"). Prends la première issue non-bloquée qui est en Backlog ou Todo.
2. Mets l'issue en "In Progress" dans Linear avant de commencer.
3. Lis context/architecture-v2.md pour le contexte technique global.
4. Implémente strictement le scope de l'issue. Le hors-scope est aussi important que le scope — ne fais rien qui y figure.
5. Commite et push après chaque étape significative (pas seulement à la fin).
6. Quand c'est fini, mets l'issue en "Done" et commente-la dans Linear avec un résumé de ce qui a été fait et les critères d'acceptance vérifiés.
7. Si tu es bloqué, commente l'issue dans Linear avec la question précise.

## Conventions

- Branches nommées d'après l'issue Linear (ex: gildasfremont/tes-17-capture-micro)
- Pull main régulièrement si tu travailles sur une branche
- Ne touche pas au dossier context/ (c'est Cowork qui y écrit)
